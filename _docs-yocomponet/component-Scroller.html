Scroller 滚动组件 3.0.0

滚动组件,用来提供滚动容器。

  • 提供了横向滚动和纵向滚动。
  • 提供了『下拉刷新』和『加载更多』功能。
  • 提供了 transition 和 requestAnimationFrame 两种实现滚动的方式。
  • 提供了 transform 和 position:absolute 两种实现位移的方式。

确定高度：Scroller 必须有一个确定的高度才能正常工作，因为它实际上就是将一系列不确定高度的子组件装进一个确定高度的容器。实现确定高度的方式有很多种：flex、指定高度、position: absolute等等。

内容容器：作为一个滚动组件，Scroller 会创建一个 div 作为滚动容器。如果 Scroller 的子元素只有一个，则会把这个子元素当做内容容器；否则，会创建一个 div 作为内容容器。

属性

extraClass { PropTypes.string }

为组件根节点提供额外的class。

默认值: ''

containerExtraClass { PropTypes.string }

为组件中的内容容器提供额外的class。

默认值: ''

contentOffset { {x: PropTypes.number, y: PropTypes.number} }

组件中内容的初始位移，这个属性变化时，会重置内容的位移。

默认值: {x: 0, y: 0}

enabled { PropTypes.bool }

是否允许滚动，默认允许。

默认值: true

scrollX { PropTypes.bool }

是否开启横向滚动，默认关闭。

默认值: false

scrollY { PropTypes.bool }

是否开启纵向滚动,默认开启。

默认值: true

directionLockThreshold { PropTypes.number }

只允许单向滚动的时候，会根据这个阀值来判定响应哪个方向上的位移：某一方向位移减去另一个方向位移超过阀值，就会判定为这个方向的滚动。

默认值: 5

momentum { PropTypes.bool }

是否允许惯性滚动。当设置为 true，手指离开时，如果还有速度会继续滚动一段距离；当设置为 false ，手指离开时会立即停止滚动。

默认值: true

bounce { PropTypes.bool }

当滚动超出内容范围时，是否可以继续滚动一截。

默认值: true

bounceTime { PropTypes.number }

当弹性滚动一截之后，回到滚动范围内位置的时间，单位为毫秒（ms）。

默认值: 600

bounceEasing { PropTypes.Object }

弹性回滚动画。

Scroller 提供了五种默认的动画函数：quadratic, circular, back, bounce, elastic，可以通过 Scroller.ease.xxx 来使用。

用户也可以自定义动画对象，比如：

{ style: 'cubic-bezier(0.25, 0.46, 0.45, 0.94)', fn: function (k) { return k * ( 2 - k ); } }

默认值: Scroller.ease.circular

useTransition { PropTypes.bool }

如果设置为true,会使用transition来实现滚动效果;如果设置为false,会使用requestAnimationFrame来实现。

默认值: true

useTransform { PropTypes.bool }

如果设置为true,会使用transform来实现位移;如果设置为false,会使用left和top来实现位移（position: absolute）。

默认值: true

onScroll { Function }

(event) => void

滚动事件的回调。一旦设置了这个回调，为了能够监听滚动事件，会将useTransition属性强制设置为false，会由此带来一定的性能牺牲。

方法参数:

参数名 类型 描述 必选 版本
event e 滚动事件的回调，结构为: {contentOffset: {x: x, y: y}}

HWCompositing { PropTypes.bool }

是否开启硬件加速

默认值: true

usePullRefresh { PropTypes.bool }

是否开启下拉刷新功能

默认值: false

onRefresh { Function }

(event) => void

下拉刷新时开始刷新的回调。

方法参数:

参数名 类型 描述 必选 版本
event e 结构为: ({contentOffset: {x: x, y: y}})

pullRefreshHeight { PropTypes.number }

触发下拉刷新状态的高度（一般即为下拉刷新提示区域的高度）

默认值: 40

renderPullRefresh { PropTypes.func }

() => JSX

自定义的下拉刷新渲染函数

useLoadMore { PropTypes.bool }

是否开启加载更多功能.『加载更多』与『下拉刷新』略有不同，加载更多的提示区域是追加在内容区域的最后

默认值: false

onLoad { Function }

(event) => void

加载更多时开始加载的回调。

方法参数:

参数名 类型 描述 必选 版本
event e 结构为: ({contentOffset: {x: x, y: y}})

loadMoreHeight { PropTypes.number }

触发加载更多状态的高度（一般即为加载更多提示区域的高度）

默认值: 40

renderLoadMore { Function }

() => JSX

自定义的加载更多渲染函数

方法

stopAnimate

停止当前的滚动动画，包括：惯性滚动、回弹、ScrollTo等。

refresh

刷新 Scroller，一般场景不推荐使用，因为当内容改变的时候，Scroller 会自动 render。

使用场景1：需要强制设置 Scroller 本身的宽高和内容容器的宽高时，可以通过refreshOption来传入宽高代替dom的宽高。

使用场景2：在某些不是通过 setState 或 Redux 等方式来改变内容导致 Scroller 不会 render 时，可以强制重新获取Scroller宽高和内容容器宽高。

方法参数:

参数名 类型 描述 必选 版本
refreshOption Object 刷新参数，{wrapperWidth, wrapperHeight, scrollerWidth, scrollerHeight}
doNotRefreshWrapper Bool 强制不刷新位移

scrollTo

滚动到某个位置。

方法参数:

参数名 类型 描述 必选 版本
x Number 水平位移
y Number 垂直位移
time Number 滚动时间
easing Object 滚动动画对象。参照 `bounceEasing` 参数。

startRefreshing

强制开始刷新。这个方法一般是用在切换筛选项或者关键字等场景，来达到回到顶部并且开始刷新的效果。如果是用户下拉触发 onRefresh 时，就不需要再调用这个方法了。

方法参数:

参数名 类型 描述 必选 版本
time Number 滚动到顶部的时间，默认为 300ms

stopRefreshing

停止刷新，停止之后会自动滚动到顶部。

方法参数:

参数名 类型 描述 必选 版本
status Bool 刷新的状态。true表示加载成功，false表示加载失败。
config Object 停止刷新的动画配置
config.duration number 回到顶部的动画时间，默认是300ms

stopLoading

停止加载更多。

方法参数:

参数名 类型 描述 必选 版本
status Bool 刷新的状态。true表示加载了更多数据，false表示没有更多数据了。
